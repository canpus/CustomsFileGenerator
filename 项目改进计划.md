## 三、模板策略

### 3.1 多容量模板方案

在 `templates/` 中为每种文件类型准备 3 种容量：

```text
template_invoice_20.xlsx   / template_invoice_50.xlsx   / template_invoice_100.xlsx
template_contract_20.xlsx  / template_contract_50.xlsx  / template_contract_100.xlsx
template_packing_20.xlsx   / template_packing_50.xlsx   / template_packing_100.xlsx
```

当前已有模板作为 50 行版本的基准。

### 3.2 自动选择规则

| 明细行数 | 选择模板 |
|-----------|----------|
| rows ≤ 20 | 20 行模板 |
| 21 ≤ rows ≤ 50 | 50 行模板 |
| 51 ≤ rows ≤ 100 | 100 行模板 |
| rows > 100 | 阻断生成，提示拆单 |

初版不自动扩行、不自动拆分文件。

### 3.3 数据区处理原则

- 通过 `template_rules.json` 显式配置数据起始行、预留结束行、汇总行、公式列
- 填充 N 行后，删除第 `start + N` 到 `end` 的预留空行，汇总行自动上移
- 不再通过"遇到第一个空行"判断数据区结束
- 表头已有字段名时只写字段值，不写 `Invoice No.: 20251202-01` 拼接文本

- DOCX 报关单生成（Stage 6 恢复，待后续规划）

## 一、项目总览

### 这个程序做什么？

你的公司需要为每笔出口订单生成 4 份报关文件：

1. **报关单**（.docx Word 文件）
2. **装箱单**（.xlsx Excel 文件）
3. **形式发票**（.xlsx Excel 文件）
4. **形式合同**（.xlsx Excel 文件）

现在可能是手工填写，这个程序的目标是：**你输入一次数据，自动生成全部 4 份文件**，省去重复劳动和人工计算错误。

### 技术栈（用什么做的）

| 技术 | 作用 | 为什么选它 |
|------|------|-----------|
| Python 3.12 | 编程语言 | 你学过的那个 |
| tkinter + ttkbootstrap | 窗口界面 | Python 自带，不需要额外装 |
| openpyxl | 读写 Excel 文件 | 不会破坏模板原有的格式 |
| python-docx | 读写 Word 文件 | 唯一选择 |
| msgspec | 校验数据是否正确 | 比 Pydantic 轻 10 倍 |
| sqlite3 | 存客户/产品/模板数据 | Python 自带，零配置 |
| PyInstaller + UPX | 打包成 .exe | 让你同事不用装 Python 也能用 |

### 程序最终长什么样？

```
┌──────────────────────────────────────────┐
│  报关资料自动生成系统          ─ □ ✕    │
├──────────┬───────────────────────────────┤
│ 📋 新建单据│  订单信息                    │
│ 📁 历史模板│  ┌─────────────────────┐    │
│ 👤 客户管理│  │ 发票号：[____]      │    │
│ 📦 产品库  │  │ 合同号：[____]      │    │
│           │  │ 日期：[____]        │    │
│           │  └─────────────────────┘    │
│           │  托盘-纸箱-商品              │
│           │  📦 托盘1                   │
│           │    📁 纸箱1-10              │
│           │      📄 商品A               │
│           │      📄 商品B               │
│           │                             │
│           │  [一键生成] [导出诊断包]     │
└──────────┴───────────────────────────────┘
```

### 开发分几个阶段？

总共 **10 个阶段**，你用 10 个独立的 Agent 对话窗口完成。每个阶段完成后你会得到一个可验证的成果。

## 三、你的电脑环境配置

### 第 1 步：安装 Python

1. 去 Python 官网下载：https://www.python.org/downloads/
2. 下载 **Python 3.12.x**（64-bit Windows installer）
3. 安装时 **务必勾选"Add Python to PATH"**（添加到环境变量）
4. 安装完成后验证：

```bash
# 打开"命令提示符"（Win+R → 输入 cmd → 回车）
python --version
# 应该显示：Python 3.12.x
```

### 第 2 步：安装 Git（可选但推荐）

从 https://git-scm.com/download/win 下载安装。

### 第 3 步：安装 VS Code（推荐编辑器）

从 https://code.visualstudio.com/ 下载安装。

推荐安装的 VS Code 插件：
- **Python**（微软官方）
- **Ruff**（代码检查工具）

### 第 4 步：创建项目文件夹

在你的电脑上创建一个文件夹，例如：

```
E:\Coding_Programs_New\CustomsFileGenerator\
```

把 4 个模板文件放入 `E:\Coding_Programs_New\CustomsFileGenerator\templates\` 文件夹。

### 第 5 步：安装 Python 依赖

Agent 会在阶段 0 生成 `requirements.txt`，你需要在项目文件夹中打开命令提示符，执行：

```bash
cd E:\Coding_Programs_New\CustomsFileGenerator
pip install -r requirements.txt
```

如果下载慢，可以用国内镜像：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 十一、阶段 6：报关单生成器（Word）

### ⚠️ 技术难度最高的阶段

Word 文件的操作比 Excel 复杂很多，因为 Word 没有"单元格坐标"，Agent 需要通过 XML 操作来克隆表格行。

### 你要做什么

新对话窗口 + 阶段 0-5 产出 + `plan_v6.md` 阶段 6 + Claude.md。

### 里程碑验证方法

```bash
python -m pytest tests/test_customs_generator.py -v
```

**✅ 成功的标志**：7 个测试全部 PASSED

**人工检查重点**（用 Word 打开生成的文件）：

1. 表头 8 行信息是否正确
2. 每个商品是否占 2 行（上行：项号/HS/名称/净重/币制；下行：申报要素/数量单位/目的国/单价/总价）
3. HS Code 是否为 Arial 10pt 加粗
4. 中文字体是否为 Simsun 小五

**❌ 常见问题**：
- 商品行格式不对（上下行混乱）→ Agent 的成对克隆逻辑有 bug
- HS Code 字体不对 → Agent 没有保留 Run 级别的字体样式
- 表格行数不对 → Agent 的克隆数量计算错误

## 十五、阶段 🔟：打包成 exe

### 你要做什么

新对话窗口 + 阶段 0-9 产出 + `plan_v6.md` 阶段 🔟 + Claude.md。

### 打包步骤

```bash
# 1. 安装 PyInstaller（如果还没装）
pip install pyinstaller

# 2. 执行打包
build_upx.bat       # 生成压缩版 exe（体积小，但可能被杀软误报）
# 或
build_standard.bat  # 生成标准版 exe（体积大一些，但不会误报）

# 3. 验证
python verify_build.py
```

### 验证方法

**在你的电脑上**：
1. 双击 `dist/CustomsFileGenerator.exe`
2. 确认 GUI 正常打开
3. 录入一份简单订单 → 生成文件

**在同事的电脑上（关键验证）**：
1. 把 exe 文件复制到同事电脑（不需要安装 Python）
2. 双击运行
3. 如果能正常打开并使用 → ✅ 打包成功

**❌ 常见问题**：
- 杀软拦截 → 使用 `build_standard.bat` 生成的无 UPX 版本
- 提示缺少 DLL → 需要在一台"干净"的 Windows 虚拟机上测试
- 文件太大（超过 50MB）→ 告诉 Agent 需要进一步精简

## 十六、常见报错与解决方法

### 环境相关

| 报错信息 | 原因 | 解决方法 |
|---------|------|---------|
| `'python' 不是内部或外部命令` | Python 没装或没添加到 PATH | 重装 Python，勾选"Add Python to PATH" |
| `ModuleNotFoundError: No module named 'openpyxl'` | 依赖没装 | 执行 `pip install -r requirements.txt` |
| `pip 不是内部或外部命令` | pip 没在 PATH 中 | 用 `python -m pip install -r requirements.txt` |
| `_tkinter.TclError: no display name` | 在无图形界面的服务器上运行 GUI | GUI 程序需要在 Windows 桌面环境下运行 |

### 模板相关

| 报错信息 | 原因 | 解决方法 |
|---------|------|---------|
| `❌ 模板文件缺失：templates/template_customs.docx` | 模板文件不在正确位置 | 确认 4 个模板文件在 `templates/` 目录下，文件名必须严格匹配 |
| `❌ 锚点扫描失败：未找到数据起始行` | 模板格式与预期不符 | 检查模板是否被修改过（行增减、列增减）。如果确认模板正确，告诉 Agent 调整锚点关键词 |
| `openpyxl 报错：merged cell range is not valid` | 合并单元格在删除行时被破坏 | 这是 Agent 的 bug，把完整报错给 Agent |

### 数据相关

| 报错信息 | 原因 | 解决方法 |
|---------|------|---------|
| `❌ 校验失败：总毛重(100.0) < 总净重(120.0)` | 数据不合理：毛重应该 ≥ 净重 | 检查录入数据，毛重包含包装重量，应该大于净重 |
| `❌ 校验失败：第 2 托盘纸箱毛重(15.0) < 箱内商品净重之和(18.5)` | 数据不合理 | 检查该纸箱的商品净重数据 |
| `❌ 缺少必填字段「发票号」` | 没有填发票号 | 在 GUI 中填写发票号 |

### 生成文件相关

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 生成的 Excel 打不开 | 文件在写入过程中损坏 | 重试生成。如果持续出现，告诉 Agent |
| 生成的 Excel 格式错乱 | 样式克隆有 bug | 检查是否是新插入的行没有正确复制样式 |
| 生成的 Word 商品行格式不对 | 成对克隆逻辑有 bug | 检查每个商品是否占 2 行、HS Code 字体是否正确 |
| 汇总数据不会自动更新 | 公式被覆盖或范围错误 | Agent 需要检查 `update_sum_formula` 函数 |

### GUI 相关

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 窗口文字模糊 | DPI 缩放未激活 | 在阶段 9 确认 Agent 实现了 DPI 感知 |
| 点击按钮没反应 | 事件绑定错误 | 把操作步骤和现象告诉 Agent |
| 界面显示不全 | 窗口尺寸不对 | 可能需要设置最小窗口尺寸 |
| 中文字体显示为方块 | 系统中没有中文字体 | 通常不会出现，因为 Agent 实现了字体回退池 |

### 打包相关

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 打包后 exe 体积超过 50MB | 打包了不必要的库 | 告诉 Agent 需要在 `.spec` 中增加 excludes |
| exe 被杀软拦截 | UPX 压缩触发了误报 | 用无 UPX 版本打包 |
| 双击 exe 闪退 | 程序启动时崩溃 | 在命令提示符中运行 exe，看报错信息 |
| `ImportError: DLL load failed` | 缺少系统运行库 | 目标电脑可能需要安装 Visual C++ Redistributable |

## 十七、附录：术语表

| 术语 | 解释 |
|------|------|
| **SSOT** | Single Source of Truth，单一数据源。所有模块用同一份数据，避免数据不一致 |
| **msgspec** | 一个 Python 库，用于校验数据结构是否正确，比 Pydantic 更快更小 |
| **WAL 模式** | SQLite 的一种工作模式，提升读写性能 |
| **参数化查询** | 防止 SQL 注入攻击的安全编程方式 |
| **锚点扫描** | 程序自动在模板中找到"数据应该从哪里开始填"的位置 |
| **成对克隆** | 报关单中每个商品占 2 行，克隆时把这两行当作一个整体一起复制 |
| **沙箱操作** | 先在临时文件夹中操作，成功了再复制到目标位置，避免损坏原始模板 |
| **脱敏** | 把敏感信息（公司名、电话、单价）替换为 `***`，保护隐私 |
| **DPI 感知** | 让程序在高分辨率屏幕上显示清晰，不会模糊 |
| **UPX** | 一个压缩工具，能把 exe 文件压缩得更小 |
| **CI** | Continuous Integration，持续集成。自动检查代码是否合格 |
| **PyInstaller** | 把 Python 程序打包成独立 exe 的工具 |
